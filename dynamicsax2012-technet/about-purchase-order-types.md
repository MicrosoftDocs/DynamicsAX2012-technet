# About purchase order types 


_**Applies To:** Microsoft Dynamics AX 2012 R3, Microsoft Dynamics AX 2012 R2, Microsoft Dynamics AX 2012 Feature Pack, Microsoft Dynamics AX 2012_

When you create a purchase order, you must indicate which type of an order it is. You can select one of the types that are described here:


> [!TIP]
> <P>You can set up a default purchase order type in the <A href="https://technet.microsoft.com/en-us/library/hh208706(v=ax.60)">Procurement and sourcing parameters (form)</A> in the <STRONG>Purchase type</STRONG> field.</P>



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th><p>Purchase order type</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p><strong>Journal</strong></p></td>
<td><p>Use as a draft. This type does not affect stock quantities and does not generate item transactions. The purchase order journal lines are not included in master scheduling.</p></td>
</tr>
<tr class="even">
<td><p><strong>Purchase order</strong></p></td>
<td><p>Use when the vendor confirms an order.</p></td>
</tr>
<tr class="odd">
<td><p><strong>Returned order</strong></p></td>
<td><p>Use when you return goods to the vendor. The returned item number, this is the RMA number, that the vendor gives you must be entered on the <strong>Other</strong> tab.</p></td>
</tr>
</tbody>
</table>
