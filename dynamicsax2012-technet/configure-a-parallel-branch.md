# Configure a parallel branch 


_**Applies To:** Microsoft Dynamics AX 2012 R3, Microsoft Dynamics AX 2012 R2, Microsoft Dynamics AX 2012 Feature Pack, Microsoft Dynamics AX 2012_

To configure a parallel branch, complete the following procedures in the workflow editor.

## Name the branch

Follow these steps to enter a name for the parallel branch.

1.  Right-click the parallel branch and then click **Properties**. The **Properties** form is displayed.

2.  In the left pane, click **Basic Settings**.

3.  In the **Name** field, enter a unique name for the parallel branch.

4.  Click **Close**.

## Design and configure the elements of the branch

A parallel branch is essentially a workflow that runs in the context of a parent workflow. Follow these steps to design and configure the elements of the parallel branch.

1.  Double-click the parallel branch.

2.  Drag workflow elements onto the canvas, and then configure the elements, just as you would to create any other workflow.
    
    For more information, see [Create a workflow](create-a-workflow.md).
