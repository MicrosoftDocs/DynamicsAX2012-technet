# Create a functional area in RapidStart Services 


_**Applies To:** Microsoft Dynamics AX 2012 R3, Microsoft Dynamics AX 2012 R2, Microsoft Dynamics AX 2012 Feature Pack, Microsoft Dynamics AX 2012_

Beginning October 1, 2015, RapidStart services will no longer be supported. For more information, see the announcement on CustomerSourcehttps://mbs.microsoft.com/partnersource/northamerica/news-events/news/RapidStartServicesDiscontinued or PartnerSourcehttps://mbs.microsoft.com/customersource/northamerica/news-events/news-events/news/RapidStartServicesDiscontinued.

In Microsoft Dynamics ERP RapidStart Services, functional areas are used to collect groups of questions. The answers to these questions that are provided by business experts in the organization are used to configure an implementation of Microsoft Dynamics AX 2012. You create functional areas in the **Design** area in RapidStart Services.

1.  In the RapidStart Services header, click **Design**, and then click **Functional area**.

2.  Click the **Functional area** button, and then click **New functional area**.

3.  In the **Functional area** form, in the **Name** field, enter a name to identify the functional area.

4.  Optional: In the **Description** field, describe the purpose of the functional area and how it will be used.

5.  On the **Release version** tab, select the check box for each version of Microsoft Dynamics AX 2012 that this functional area can be used to configure.

6.  On the **Industry** tab, select the industries to which this functional area applies.

7.  On the **Country/region** tab, select the countries/regions to which this functional area applies.

8.  Click **OK**.
