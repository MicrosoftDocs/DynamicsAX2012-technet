# Move a question group in RapidStart Services 


_**Applies To:** Microsoft Dynamics AX 2012 R3, Microsoft Dynamics AX 2012 R2, Microsoft Dynamics AX 2012 Feature Pack, Microsoft Dynamics AX 2012_

Beginning October 1, 2015, RapidStart services will no longer be supported. For more information, see the announcement on CustomerSourcehttps://mbs.microsoft.com/partnersource/northamerica/news-events/news/RapidStartServicesDiscontinued or PartnerSourcehttps://mbs.microsoft.com/customersource/northamerica/news-events/news-events/news/RapidStartServicesDiscontinued.

In Microsoft Dynamics ERP RapidStart Services, you can move a question group from one functional area to another. If the question group that you are moving has dependent questions, you are prompted to resolve any issues before the question group is moved.


> [!NOTE]
> <P>You cannot move a question group that belongs to a functional area that has been published. For information about how to unpublish a functional area, see <A href="unpublish-a-functional-area-in-rapidstart-services.md">Unpublish a functional area in RapidStart Services</A>.</P>



1.  In the RapidStart Services header, click **Design**, and then click **Functional area**.

2.  In the **Functional area** list, select the functional area that contains the question group that you want to move.

3.  Click the **Functional area** button, and then click **Move question group**.

4.  In the **Move question group** form, in the **Target functional area** list, select the name of the functional area to which to move one or more question groups.

5.  Below the **Source functional area** field, select the name of the question group that you want to move, and then click **Add \>\>**.

6.  Repeat step 5 for each question group that you want to move.

7.  Click **Next**.

8.  Check your selections on the review screen.

9.  Click **Finish**.
