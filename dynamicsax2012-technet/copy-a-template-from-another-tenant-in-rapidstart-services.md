# Copy a template from another tenant in RapidStart Services 


_**Applies To:** Microsoft Dynamics AX 2012 R3, Microsoft Dynamics AX 2012 R2, Microsoft Dynamics AX 2012 Feature Pack, Microsoft Dynamics AX 2012_

Beginning October 1, 2015, RapidStart services will no longer be supported. For more information, see the announcement on CustomerSourcehttps://mbs.microsoft.com/partnersource/northamerica/news-events/news/RapidStartServicesDiscontinued or PartnerSourcehttps://mbs.microsoft.com/customersource/northamerica/news-events/news-events/news/RapidStartServicesDiscontinued.

This topic explains how to copy a template from another tenant as the basis for a new template in Microsoft Dynamics ERP RapidStart Services. After you create the copy, you can modify the copied functional areas, question groups, and questions in order to customize your own implementation of Microsoft Dynamics AX.


> [!NOTE]
> <P>To copy a template from another tenant, you must be assigned to the Viewer role for your organization, in addition to your role as a Designer or Configurator. For information about role assignments in RapidStart Services, see <A href="add-users-and-assign-roles-in-rapidstart-services.md">Add users and assign roles in RapidStart Services</A>.</P>



To create a new template by copying an existing one from another tenant, follow these steps:

1.  In the RapidStart Services header, click **Design**, and then click **Templates**.

2.  Click the **Template** button, and then click **Copy template from**.

3.  In the **Copy template** form, select the check box next to the name of the template that you want to copy.

4.  In the **NewName** field, enter the name that you want to assign this template in the tenant that you are currently working in.

5.  Click **Copy**.

Modify the copied template by adding or removing functional areas or sub-templates.
